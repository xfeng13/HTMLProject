Frequently Asked Questions for HTML Project

Q #1. I activated my www folder,  but when I check my personal webpage by typing my URL, it said it was invalid (or "page not found").

Answer: Make sure you have web hosting turned on at ASU and that your .html files are INSIDE the www folder of your AFS space. Also, be sure you typed the address correctly.

https://www.public.asu.edu/~yourASURiteID

Q #2. When making the web page, what program do we have to use? Can we use FrontPage, Dreamweaver, etc? Or do we have to use Notepad?

Answer: You have to use a plain text editor (other than empty spaces, without any formatting features), or any text editors which allow you to save the index.html as a plain text file. You need to write the HTML code by yourself and you can't use a HTML generator to write the HTML for you. FrontPage, Dreamweaver, Wix are all HTML generators which you can NOT use. We will check your HTML code manually to make sure it is not generated by a program.

Q #3. Where to find the text editor on my PC (or a Mac)?

  • If you're using a Windows PC, click on the "Search" box, then enter "Notepad" to open it;

  • If you have a Mac, you can use TextEdit to write the .html code. Note: TextEdit will save any .html file as a web page by default. In order to use it properly, you need to turn off its fancier options, click on the following link to see instruction on how to do this. Note: Mac users, you must do this in order to write and edit your web pages.

          http://www.radford.edu/~rstepno/326/textedit/

  • No matter which text editor you use, you must actually write the HTML code, you cannot use Frontpage, Word (oh please no, especially not that), Dreamweaver or any other program to write the HTML for you, we will check your source code for these. You need to create the file of type "plain text" and give them the file extension .html.

Q #4. I added a picture to my page but the picture doesn't come up. it just has a little box with a red x in the middle of the box. Any suggestions as to what I am doing wrong?

Answer: Several possible reasons for above error:

4.1) Each image is a file with file extension like .jpg, jpeg or .gif etc. You will need to upload each of the image file inside your www folder. For example, you have two image files, once is called ASULogo.jpg, another one is called CSE180.JPG which you want to put on your website, then you should add the following line inside your index.html file, and also put the ASULogo.jpg and CSE180.JPG inside the www folder also. Note: option width="300" or height="200" used to control the size of the image, you should adjust it according to your actual size or design.

<img src="ASULogo.jpg"  width="300" />

<img src="CSE180.JPG" height="200" />

4.2) Your image file name might be incorrect. Image file name is case sensitive and cannot contain empty spaces, dashes (-) and other special characters (like %, &), underscore is okay. For above example, if you notice that your image file is ASULogo.JPG, you must change the first line to:

<img src="ASULogo.JPG"  width="300" />

4.3) Your image file's path might be wrong. Some students said that when they view their web pages on their own computer, they can view everything, but when they upload the index.html code into www folder, the picture doesn't show up. This was caused by the image file's wrong path. For example if you put the following line inside your index.html file, even after you update the index.html, the picture won't show up, since the path to the picture file is local to your own computer and other people can't view it.

 <img src="C:\Users\feng\Documents\Sherry\CSE180\ASULogo.JPG"  width="300" />

To fix above error, you will need to change above to:

<img src="ASULogo.jpg"  width="300" />

4.4) Check your HTML tags carefully, make sure you don't have any typos. For example, if you type <img scr="ASULogo.jpg /> (instead of src), the image won't show up also.

4.5) You should NOT put a web image inside your index.html directly. For example: if you have the following line of code inside the index.html

<img src="http://clipart-for-free.blogspot.com/2008/08/smiley-face-clip-art.jpg" width="100" />

It's better that you right-click on above image and save and upload it onto your www folder as smiley_face.jpg, then change above code to:

<img src="smiley_face.jpg" width="100"/>

4.6) Mac users, be careful of the curly double quotes surrounding the image files. Read Q #5 to solve it.

Q #5. I'm a Mac user and I wrote my html code using TextEdit, but the picture doesn't show up. I checked my code and I found my picture file name was surrounded by strange charaters (see below), what went wrong?

 

<img src = ‚ÄsampleImage.jpg‚Äù  width=400 height=400 />

 

Answer: The strange characters around your picture file's name are curly double quotes which are not plain. You will need to delete the pair of the double quotes around your image file and replace them by plain double quotes.

 

To use the plain double quotes, go to System Preferences->Keyboard->Text->, then uncheck the "Use smart quotes and dashes" option. Or under TextEdit, select all your text by pressing Cmd+A and then go to Edit -> Substitutions and uncheck "Smart Quotes". Make sure Smart Quotes is not active.

 

Below please find a website that show how to use the "Dumb" quotes (plain ones), not the "Smart" ones (curly ones). 

https://www.tekrevue.com/tip/textedit-plain-text-mode/

Q #6 I'm a Mac user and I wrote my code with Mac TextEdit. I uploaded the index.html file into the www folder, but when I went to my webpage, it only displayed the actual HTML code.

Answer: Mac TextEdit uses some kind of rich text formatting by default that will cause your web page to show up as essentially a page of text of the code you wrote, so you need to change that setting on TextEdit. Click on the following link to view detailed explanation and methods on how to fix this:

 

https://www.tekrevue.com/tip/textedit-plain-text-mode/

 

Q #7. I did everything correctly and uploaded my index.html onto the website, but when I check my website, it still only display the default "Welcome..." page, what did I do wrong?

Answer:  7.1) First, check your index.html file's name, it MUST be index.html (all lower cases), if you name it as Index.html, index2.html, index.txt.html, index.html.txt or myIndex.html, you will only see the default "Welcome.." page.

 

7.2) You need to put the index.html file INSIDE the www folder. You will need to double click the www folder to open it, then upload your newest index.html file inside to replace the old one.

 

7.3) I recommend that you delete your original index.html first, then upload your newest/updated index.html inside the www folder. Sometimes if you just upload the updated index.html file inside, the older one doesn't get replaced successfully. 


Q #8. How is the resume.html supposed to be done? Is the entire resume typed and inserted into our notepad or can I used Microsoft Word to help with fonts and colors, and then pasted those HTML codes to Notepad, is that acceptable?

Answer:  No. Do not use any program other than a text editor for this assignment. You can look up HTML tags for fonts and colors on the web. You have to type your resume in notepad, and it must be an HTML page. You should design your resume.html similarly as you designed the index.html.

 

Q #9. I have linked my index.html page to my resume page, but it doesn't link correctly. What am I doing wrong?

Answer: To add a link in resume.html that links back to your home page, add the following line, for example, inside resume.html
 

Click here to go back to my <a href= "index.html">home page</a>
 

Modify your index.html such that it includes a hyperlink that links back to your resume page. To do this, add the following line inside index.html.


Click here to see <a href="resume.html">my resume</a>

 

Be careful about the path of the link, if you have, for example, the following line inside your index.html file, then the link doesn't work since it links to a file on your local hard drive C, not a directly file inside the www folder.

 

Click here to see <a href="C:\Users\feng\Documents\Sherry\CSE180\resume.html">my resume</a>

 

Q #10. My webpage keeps displaying the HTML tags as they appear in my text file instead of the page as it should be displayed in a browser. What should I do?

Answer: That means one or more of your tags contain errors or are incorrect, resulting in the web browser does not see it as HTML. Double check your index.html or resume.html again, make sure are tages are paired properly and you don't have any typos in them. 

Q #11. After using a text editor to write our html file, how do we save it as an html file?

Answer: Click on "Save as", and save your file as "index.html" and make sure to change your text editor's options to save as type "all files".

Q #12. Whenever I save changes to my index.html file, my changes do not get saved, and my website appears unchanged online. What am I doing wrong?

Answer: "Saving" the file on your hard drive does not update it on the web server. You need to upload the updated file to replace your original index.html file inside the www folder. Sometimes you might need first delete your index.html file inside the www folder, then upload the latest one.

  

Q #13. I want to make some changes to my original index.html file, I downloaded it from www folder onto my hard drive, how to re-open it as plain text file and edit it from there?

Answer: Make sure you know where you download the index.html file is, then open the plain text editor, such as Notepad or TextEdit, from the top menu, pick "File" => "Open", then locate the index.html to open it. After you make any changes to the index.html file, make sure save them.

 

Q #14. Is it all right to use the "My Files" under "My ASU" to upload the index.html and resume.html files?

Answer: Yes, that is fine. Just make sure to upload it to your "www" folder. Be careful! Many students accidentally deleted their www folder, or upload files to the wrong folder.

Q #15. I was wondering if we were allowed to use any CSS or if it strictly has to be HTML.

Answer: You should use strictly HTML for the assignment. After the assignment is graded, you can update your site in any way you wish.
